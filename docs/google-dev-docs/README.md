# Google Search Central docs — local knowledgebase

Local, grounded copies of the full Google developer documentation tree on crawling
and indexing, so agents/developers can read authoritative context without
re-fetching. Covers both doc sections (62 pages, fetched 2026-07-03):

- <https://developers.google.com/search/docs/crawling-indexing> — Search Central:
  crawling & indexing
- <https://developers.google.com/crawling> — Crawling infrastructure
  (crawlers-fetchers, robots-txt, troubleshooting; Google is moving the robots.txt
  and crawler pages here, and old `/search/docs/crawling-indexing/...` URLs 301 to
  the new homes — files here are named after the *canonical* post-redirect URL)

Generated by `tools/google-dev/discover-pages.sh` (enumerates the tree) and
`tools/google-dev/pull-docs.sh` (fetches + extracts):

- `<slug>.md` — extracted text with a source/license attribution header. **Committed.**
- `rawhtml/<slug>.html` — the raw fetched page. **Gitignored**; rerun the script to
  rematerialize.

## Regenerating / refreshing

```sh
# refresh the whole knowledgebase (discover the current doc tree, pull every page):
tools/google-dev/pull-docs.sh                 # no args = discover + pull

# equivalent, with the two stages explicit:
tools/google-dev/discover-pages.sh | tools/google-dev/pull-docs.sh -

# pull specific page(s) only:
tools/google-dev/pull-docs.sh https://developers.google.com/crawling/docs/robots-txt/robots-txt-spec
```

`discover-pages.sh` BFS-crawls the section navigation embedded in the devsite HTML,
follows redirects to canonical URLs, strips anchors/locale params, and emits one URL
per line. `pull-docs.sh` fetches each URL (also redirect-canonicalized, so a moved
page lands under one slug), saves the raw HTML to `rawhtml/`, and extracts attributed
markdown. After a refresh: if Google moved pages, delete committed `.md` files whose
slug no longer appears in the discovery output, and update the index above (the
[changelog page](crawling-docs-changelog.md) summarizes what changed on Google's side).
